基于HEXO搭建博客复盘

发表于 2021-09-21 更新于 2021-09-23 分类于 学习 ， 编程 ， 前端

搭建博客算是我自接触CS以来第一次独立完成的项目，实现了我在学C道路上从0到1的跨越。在此我将对个人搭建过程进行细致的复盘，在将其强化我自身知识储备的同时，也能为看到这篇博文的各位提供一些有价值的帮助。

第一部分 准备与环境配置

运行环境

Windows 10

准备工具

Git

简介：Git是一个开源的分布式版本控制系统，用以有效、高速的处理从很小到非常大的项目版本管理。
安装教程：参考此博客
PS: 可直接从官网下载，从第二步开始参考步骤

VScode

简介：VSCode 全称 Visual Studio Code，是微软出的一款轻量级代码编辑器，免费、开源而且功能强大，一般为前端代码开发IDE。
安装教程：参考此文

NodeJS

简介：Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。Node.js 使用了一个事件驱动、非阻塞式 I/O 的模型，使其轻量又高效。Node.js 的包管理器 npm，是全球最大的开源库生态系统。
安装教程：从官网下载自己电脑对应的LTS版本，直接安装。

第二部分 正式步骤

1. 打开CMD（即命令提示符）

你可以在开始界面找到名为Wimdows系统的文件夹，命令提示符就在文件夹内，也可以快捷键windows加R键呼出，务必记住使用管理员身份运行。

2. 安装淘宝镜像源

在CMD中输入以下代码
$ npm install -g cnpm --registry=https://registry.npm.taobao.org
检测是否安装成功
输入cnpm -v查看版本信息

3. 使用cnpm安装HEXO

在CMD中输入以下代码
$ cnpm install -g hexo-cli
检测是否安装成功
输入hexo -v查看版本信息

4. 建立blog文件夹

随便你建在哪里，桌面也行。以后所有东西都将放在此文件夹运行使用，不要畏惧，出了问题就删掉整个文件夹重来。

5. 初始化一个博客

进入blog文件夹，右键点开Git Bash Here。输入hexo init
（如果提示无法安装说明未启用管理员模式，此时可选择cmd安装，输入同样的指令）

6. 查看本地化的HEXO是否完成

同上一步一样打开Git，输入hexo s,然后打开网址localhost:4000，能看到初始化的网站，便代表博客已经安装成功。

第三部分 部署HEXO到GitHub上

1. 注册GitHub账号

HEXO官网

2. 建立新仓库

仓库名的格式是 你的GitHub名字.github.io

3.生成ssh密钥

在本地博客文件夹中右键选择Git Bash Here调出命令行，输入以下命令：
git config --global user.name "YourGithubName" //双引号内填写你的GitHub用户名
git config --global user.email "YourGithubEmail" //双引号内填写你的Github个人邮箱

输入以下命令生成SSH密钥：
ssh-keygen -t rsa -C "YourGithubEmail" //双引号内填写你的Github个人邮箱

输入以下命令将其复制：
cat ~/.ssh/id_rsa.pub //在Git命令提示符中显示出SSH密钥

接下来打开Github，点击头像在选项卡中找到Setting设置选项，再点击SSH and GPG keys，新建一个SSH，命名随意，将刚才再Git输入框中复制的密钥粘贴在此处，点击保存。
回到Git命令行，输入以下命令：
ssh -T git@github.com //查看本地是否与Github连接成功

4.修改Hexo配置文件

站点配置文件本地地址如下：
/hexo/themes/landscape/_config.yml
打开站点配置文件，修改配置信息：

5.上传到GitHub

在git中输入hexo d

(此命令表示上传，以后更新博客会经常用到)

输入GitHub账号与密码

输入完成后等待GitHub上传与创建

6.登入你的博客网址

至此，基于hexo建立的个人博客已经完成。
